# UI 规格说明 (UI_SPEC)

> 本文档定义 3D 展示幕墙 AI 专家系统的前端 UI 详细规格。

---

## 1. 设计原则

### 1.1 核心理念

- **项目驱动**：主界面是项目列表和项目详情，不是聊天框
- **结构化录入**：通过表单和向导引导用户输入，不是自由对话
- **文档编辑**：策划案页面是文档编辑器形态，不是对话气泡
- **Agent 辅助**：AI 助手在侧边栏辅助，不占据主界面

### 1.2 不做的事

- ❌ 不要做成 ChatGPT 式单页面
- ❌ 不要做成娱乐化生图社区
- ❌ 不要做成简陋的表单后台
- ❌ 不要把 AI 对话框作为唯一交互方式

### 3.2 创建项目向导

**路径**：`/workspace/projects/new`

**布局**：步骤条 + 表单区

```text
┌────────────────────────────────────┐
│  ① 企业信息 → ② 项目需求 → ③ 场地屏幕 │
│       → ④ 视觉偏好 → ⑤ 上传资料        │
├────────────────────────────────────┤
│                                    │
│  当前步骤表单                       │
│                                    │
│  [字段1] _______________           │
│  [字段2] _______________           │
│  [字段3] _______________           │
│                                    │
│           [上一步]  [下一步]         │
└────────────────────────────────────┘
```

**Step 1：客户企业信息**

- 企业名称（必填）
- 所属行业（下拉选择 + 自定义）
- 联系人
- 联系电话
- 联系邮箱
- 企业网站
- 企业简介（富文本）
- 上传企业资料（多文件）

**Step 2：项目需求**

- 项目名称
- 项目类型（下拉：裸眼3D / LED媒体立面 / 数字展示 / 其他）
- 项目目标（多选：品牌推广 / 产品发布 / 活动策划 / 展览展示 / 其他）
- 项目描述
- 预期交付时间
- 预算范围
- 特殊要求

**Step 3：场地与屏幕信息**

- 安装位置
- 屏幕类型
- 屏幕尺寸（宽 x 高）
- 分辨率
- 观看距离
- 主观看点
- 安装环境（室内/室外/半室外）
- 场地照片上传

**Step 4：视觉偏好**

- 偏好风格（多选卡片：科技感 / 奢华 / 自然 / 动感 / 简约 / 其他）
- 色调偏好
- 参考图上传
- 禁忌元素
- 其他偏好说明

**Step 5：上传资料**

- 拖拽上传区域
- 支持格式：PDF、PPT、Word、图片
- 已上传文件列表
- 文件说明备注

- 待补充信息用醒目的警告色标记

## 4. UI 风格指南

### 4.1 色彩系统

```text
主色：
  - Primary: #1E3A5F (深蓝)
  - Primary Hover: #2D5A8E

强调色：
  - Accent: #00D4FF (电光蓝/青色)
  - Accent Hover: #33DDFF

背景：
  - Page Background: #F5F7FA (浅灰白)
  - Card Background: #FFFFFF
  - Sidebar Background: #1A1A2E (深色)

文字：
  - Primary Text: #1A1A2E
  - Secondary Text: #6B7280
  - Disabled Text: #9CA3AF

状态色：
  - Success: #10B981
  - Warning: #F59E0B
  - Error: #EF4444
  - Info: #3B82F6
```

### 4.2 组件风格

- 卡片：白底、轻阴影（`shadow-sm`）、圆角（`rounded-lg`）
- 按钮：主次分明，主按钮填充色，次按钮描边
- 表单：标签在上方，输入框干净利落
- 标签/Tag：圆角胶囊形，色彩区分状态
- 表格：斑马纹，操作列右对齐

### 4.3 排版

- 标题：Inter / 系统默认 Sans-serif
- 正文：16px 基准
- 行高：1.6
- 代码/Prompt：等宽字体

### 4.4 动效

- 页面切换：淡入淡出
- 状态变化：颜色过渡
- 加载：骨架屏优于 Spinner
- 生成中：进度条 + 状态文字

## 5. 响应式要求

MVP 阶段主要面向桌面端（1280px+），但需要保证在 1024px 以上可用。

最小宽度：1024px
推荐宽度：1440px+

## 6. 组件复用清单

以下组件需要在多个页面复用，必须抽象为独立组件：

```text
StatusTag — 状态标签（颜色 + 文字）
ProjectCard — 项目卡片
FileUploader — 文件上传组件（拖拽 + 列表）
RichTextEditor — 富文本编辑器
PromptEditor — Prompt 编辑器（高亮变量）
ImageCard — 图片展示卡片（含操作按钮）
AgentPanel — Agent 助手侧边面板
ChecklistItem — 审核检查项
WizardForm — 向导表单容器
DataTable — 数据表格（排序 + 筛选 + 分页）
```
